Cross-cluster search template

This template manages remote connections for running Elasticsearch queries across multiple deployments and indices. These federated searches make it possible to break up large deployments into smaller, more resilient Elasticsearch clusters. You can organize deployments by departments or projects for example, but still have the ability to aggregate query results and get visibility into your Elasticsearch Service infrastructure. You can add remote connections either when you create your deployment or when you customize it. To know more about cross-cluster search, see Enable cross-cluster search.

Included in this template:

Amazon Web Services (AWS)
  • Elasticsearch cross-cluster search nodes: Start at 1 GB memory x 1 availability zone. Hosted on AWS r4 instances.
  • Kibana: Starts at 1 GB memory x 1 availability zone. Hosted on AWS r4 instances.
Google Cloud Platform (GCP)
  • Elasticsearch cross-cluster search nodes: Start at 1 GB memory x 1 availability zone. Hosted on custom memory-optimized GCP instances.
  • Kibana: Starts at 1 GB memory x 1 availability zone. Hosted on custom memory-optimized GCP instances.
Microsoft Azure
  • Elasticsearch cross-cluster search nodes: Start at 1 GB memory x 1 availability zone. Hosted on Azure E32sv3 instances.
  • Kibana: Starts at 1 GB memory x 1 availability zone. Hosted on Azure E32sv3 instances.
