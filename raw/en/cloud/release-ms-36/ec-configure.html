Configuring your deployment

The information in this section covers:

  • Plan for production - Plan for a highly available and scalable deployment.
  • Configure your deployment - Customize your cluster through a full list of settings.
  • Configure Beats and Logstash with Cloud ID - Customize data shippers and data processing pipeline.
  • Enable Kibana - Explore your data with the Elastic Stack visualization platform.
  • Enable Monitoring - Monitor your cluster’s health and performance.
  • Enable Alerting - Watch for changes or anomalies in your data.
  • Enable cross-cluster searches - Run queries across several deployments.
  • Upgrade versions - Stay current with the latest Elastic Stack versions.
  • Delete your deployment - No undo. Data is lost and billing stops.
  • Elasticsearch transport module (Deprecated) - Use the Java transport client instead.
