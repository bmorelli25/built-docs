Log into the Cloud UI

To access the Cloud UI in a web browser:

  1. Connect to one of the URLs provided at the end of the installation process on your first host, replacing FIRST_HOST with the correct IP address or DNS hostname.

    http://FIRST_HOST:12400
https://FIRST_HOST:12443

    You are prompted with a warning in your browser. Ignore it and proceed to the Cloud UI. To avoid receiving warnings in the future, add your own TLS/SSL security certificates. If you can’t access the Cloud UI, check if the URL points to a private IP address.

  2. Log in as user admin with the credentials provided.
  3. Agree to the software license agreement to continue. You can opt out of sharing some basic usage statistics with Elastic.

Three deployments are always shown in the Cloud UI:

  • admin-console-elasticsearch: Backs the Cloud UI itself.
  • logging-and-metrics: Collects logs and performance metrics for your ECE installation only. You must not use this deployment to index monitoring data from your own Elasticsearch clusters or use it to index data from Beats and Logstash. Create a separate monitoring deployment for your own use.
  • security: Stores all security-related configurations.
