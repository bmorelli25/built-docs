Configuring Topbeat to Use Logstashedit

If you want to use Logstash to perform additional processing on the data collected by Topbeat, you need to configure Topbeat to use Logstash.

To do this, you edit the Topbeat configuration file to disable the Elasticsearch output and use the Logstash output instead:

output:
  logstash:
    hosts: ["127.0.0.1:5044"]

    # configure logstash plugin to loadbalance events between
    # configured logstash hosts
    #loadbalance: false

In this configuration, hosts specifies the Logstash server and the port (5044) where Logstash is configured to listen for incoming Beats connections.

To use this configuration, you must also set up Logstash to receive events from Beats.
